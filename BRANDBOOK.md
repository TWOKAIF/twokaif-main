# ТУКАЙФ — Брендбук

Источник правды для всех проектов: сайт, анкеты, боты, PNG-приложения, презентации, тексты.
Любой клод/человек, начинающий новый проект, открывает этот файл первым.

> **Принцип:** Apple-esque минимализм — белое пространство, тонкая типографика, ноль лишнего. Если сомневаешься — убирай.

**Базовый viewBox:** `0 0 100 100` или `-3 18 81 81` (центрирован)

**Варианты:**
- На светлом фоне → `#1d1d1f`
- На тёмном фоне → `#ffffff`

|---|---|---|
| `--bg` | `#ffffff` | Основной фон |
| `--surface` | `#f5f5f7` | Карточки, секции с подложкой |
| `--surface2` | `#e8e8ed` | Вторичные карточки, кнопки-бейджи |
| `--text` | `#1d1d1f` | Основной текст, заголовки |
| `--muted` | `#86868b` | Подзаголовки, описания, второстепенный текст |
| `--border` | `rgba(29,29,31,.08)` | Тонкие разделители |
| `--border-row` | `rgba(29,29,31,.16)` | Бордеры контуров кнопок, табличных строк |
| `#0a0a0a` | — | Тёмный режим / theme-color dark / тёмные UI-инверсии |

### Акцент (если нужен)

Базовый акцент TWOKAIF — **отсутствие акцента**. Контраст и иерархию делаем чёрным/серым.
Если проект требует фирменного цвета (PNG-приложение клиента, лендинг под событие) — выбираем **один** акцентный цвет на проект и используем его умеренно.

### Selection
```css
::selection { background: rgba(29, 29, 31, 0.15); }
```

### Запрещено
- Градиенты в UI (логотипе, кнопках, карточках). Градиент допустим только в художественной графике (обложка, афиша).
- Полупрозрачные «стеклянные» подложки (glassmorphism).
- Тени-обводки больше 2 уровней. Обычно — 1 или 0.
- Эмодзи как смысловой акцент в интерфейсе (✓ ❌ ⚠ 🔥). Только SVG-иконки.

## 3. Типографика

### Шрифт: Inter Tight

Self-hosted в `/fonts/inter-tight.css`. Вес 300–700, вариативный. Поддержка cyrillic + latin + greek + vietnam.

**Никаких других шрифтов** в UI/документации/коммуникациях. Исключение: PNG-генератор клиента может содержать любые шрифты — это рабочий инструмент эксперта, а не интерфейс TWOKAIF.

### Шкала размеров (десктоп ≥1200px)

| Токен | Размер | Где |
|---|---|---|
| `--text-hero` | 140px | Главный экран, первый h1 на лендинге |
| `--text-display` | 80px | Большие заголовки секций |
| `--text-headline` | 56px | h2 секций |
| `--text-title` | 32px | Заголовки карточек, h3 |
| `--text-body-lg` | 24px | Лид-параграфы, цитаты |
| `--text-body` | 19px | Основной текст |
| `--text-caption` | 16px | Подписи, второстепенный текст |
| `--text-micro` | 14px | Мелкие метки, юр-футер, навигация |

### Брейкпоинты (шкала уменьшается на каждом)

- **≤1200px** — hero 100, display 64, headline 48, title 28, body-lg 22
- **≤960px** — hero 72, display 48, headline 36, title 24, body-lg 20
- **≤640px** — hero 52, display 40, headline 30, title 22, body-lg 18, body 17, caption 15
- **≤480px** — hero 44, display 32, headline 26, title 20, body-lg 17, body 16

### Веса
- **300** — крупные хедеры (hero / display) для эффекта «лёгкости»
- **400** — обычный текст
- **500** — кнопки, подзаголовки, активные пункты меню
- **600** — акцентные слова в тексте, навигация
- **700** — редко, только если очень нужно (логотип-текст, некоторые лейблы)

### Letter-spacing
- Hero/display: `-0.04em` … `-0.06em`
- Headline/title: `-0.02em` … `-0.03em`
- Body: `-0.2px` … `-0.3px`
- Caps/uppercase лейблы: `+0.5px` … `+1px`

### Line-height
- Hero/display: `0.95` … `1.0` (плотные, ударные)
- Headline: `1.05` … `1.15`
- Body: `1.4` … `1.5`
- Caption: `1.3`

### Tex-wrap
- Заголовки → `text-wrap: balance` (равномерные строки)
- Параграфы → `text-wrap: pretty` (без сирот)
- Не использовать `hyphens: auto` по умолчанию — только если кнопка маленькая и слово реально не влезает

## 4. Сетка и отступы

### Edge (отступ от края экрана)
| Брейкпоинт | `--edge` |
|---|---|
| Desktop >1200px | 48px |
| ≤1200px | 32px |
| ≤960px | 24px |
| ≤640px | 20px |
| ≤480px | 16px |

### Gap между элементами
- `--gap` = 24px (стандартный)
- Между секциями: 80–160px по вертикали (зависит от плотности контента)
- Внутри карточек: 16–24px
- Между inline-элементами: 8–12px

### 8-pt scale
Все размеры (паддинги, гэпы, радиусы) — кратные 8: `8 / 16 / 24 / 32 / 40 / 48 / 64 / 80 / 96 / 120 / 160`. Исключения — типографика и фиксированные техн. размеры (1px бордеры).

### Радиусы
- Кнопки: 0 (прямоугольные, как Apple) или 12px (карточки-кнопки)
- Карточки-секции: 16–24px
- Большие медиа-блоки: 24–32px
- Аватарки/чипы: круг (`50%`)
- **Не использовать** радиусы 4px и 8px — выглядят «вебовско-2010».

## 5. Иконки

**Только inline SVG в стиле Apple SF Symbols.**

```html
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor"
     stroke-width="1.6" stroke-linecap="round" stroke-linejoin="round">
  <!-- path -->
</svg>
```

### Параметры
- `viewBox="0 0 24 24"` базовый
- `stroke-width`: 1.6–2 (тонкая линия)
- `stroke-linecap="round"`, `stroke-linejoin="round"` — скруглённые концы
- `fill="none"` (line icons), редко `currentColor` для filled
- Цвет — `currentColor`, наследуется от родителя
- `opacity` 0.5–0.7 — для подзагрузочных меток / 1.0 — активные кнопки

### Размеры
- Inline в тексте: 16×16
- Кнопки/тулбар: 18×18 или 20×20
- Иконки секций: 24×24 или 32×32
- Hero-иконки: 48×48 / 64×64

### Источники вдохновения
Lucide / Phosphor / Tabler. Можно перерисовывать руками по мотивам SF Symbols.

### Запрещено
- Эмодзи в UI (✓ × ⚠ → 🟢 и т.д.) — рендерятся по-разному на разных ОС
- Filled-иконки с большим количеством деталей
- Цветные/градиентные иконки
- «Двухкружковые» иконки (две перекрывающиеся окружности — выглядит дёшево)
- Heroicons solid (плотные filled) — конфликтуют с минимализмом

## 6. Кнопки и формы

### Главная кнопка — `btn-fill`

```css
.btn-fill {
  display: inline-flex; align-items: center; justify-content: center; gap: 10px;
  padding: 16px 40px;
  background: var(--text); color: #fff;
  border: 1px solid var(--text);
  font: 500 15px/1 var(--font-body); letter-spacing: -0.3px;
  position: relative; overflow: hidden;
  transition: color .5s var(--ease-out);
}
/* hover — заливка слева */
.btn-fill::before {
  content:''; position:absolute; inset:0;
  background: var(--bg);
  transform: scaleX(0); transform-origin: left center;
  transition: transform .6s var(--ease-out);
}
.btn-fill:hover::before { transform: scaleX(1); }
.btn-fill:hover { color: #1d1d1f; }
.btn-fill:active { transform: scale(0.97); }
```

### Контурная — `btn-ghost`

Та же геометрия, только: `background: transparent; color: var(--text); border: 1px solid var(--border-row)`.
На hover — заливается чёрным снизу вверх.

### Магнитный hover

```html
<button class="btn-fill magnetic">…</button>
```
Кнопка с классом `magnetic` слегка тянется к курсору в радиусе ≈80% своего размера, максимум 4px смещения, коэффициент 0.06. Скрипт уже есть в `01_Глобальные-стили.html`.

### Состояния
- `:hover` — описано выше
- `:active` — `transform: scale(0.97)` (мгновенный «клик»)
- `:focus-visible` — `outline: 2px solid var(--text); outline-offset: 2px;` (всегда оставлять для доступности)
- `:disabled` — `opacity: 0.4; pointer-events: none;`

### Запрещено в кнопках
- Тени `box-shadow` (кроме фокуса)
- Скруглённые углы 4px / 8px (либо 0, либо 12+)
- Иконки + текст без `gap` (всегда 8–12px между ними)
- Текст внутри `<a>` без `text-decoration: none`

### Формы (input)
- Высота 48–56px
- Padding 14–20px
- Border 1px var(--border-row), на focus → var(--text)
- Background — белый, не серый
- Placeholder — var(--muted), без эмодзи и подсказок-смайликов
- Лейбл сверху мелким (caption 14px), не плейсхолдером

## 7. Анимации

### Easing-кривые (использовать только эти)

```css
--ease:        cubic-bezier(0.4, 0, 0.2, 1);     /* стандартный hover/transition */
--ease-out:    cubic-bezier(0.19, 1, 0.22, 1);   /* появление, заливка кнопок */
--ease-spring: cubic-bezier(0.16, 1, 0.3, 1);    /* эффект «прибытия» при скролле */
```

### Длительности
- Микро (hover, цвет, opacity): **0.3–0.5s**
- Кнопочные эффекты (заливка): **0.5–0.6s**
- Появление секций: **0.9s**
- Анимация по словам: 0.6s + delay 0.04s между словами
- **Никогда** не делай transition `0.1s` (выглядит дёрганно) или `1.5s+` (тормоза)

### Появление секций (`section-reveal`)
Уже встроено в `01_Глобальные-стили`: `opacity: 0; translateY(30px)` → срабатывает через IntersectionObserver при 12% видимости.

### Курсор-снежинка
Десктопный кастомный курсор — крутящаяся ТУКАЙФ-звезда. `mix-blend-mode: difference` — автоматически инвертируется на цвета фона. Скрывается на тач-устройствах. Уже встроено.

### Принцип
Анимация служит контенту, а не наоборот. Если можно показать что-то без анимации и оно понятно — лучше без. Анимация = подчёркивание иерархии, а не украшение.

## 8. Тон оф войс

### Главное правило
**Всегда на «ты»**. Никаких «вы», «уважаемый», «здравствуйте». Только «привет», «давай», «слушай», «если что — пиши».
Это касается ВСЕХ текстов: лендинги, боты, FAQ, push-уведомления, error states, юр-футер (кроме реквизитов ИП).

### Голос Руслана
- Дружеский, по-человечески
- Без официоза и без штампов («качественный сервис», «индивидуальный подход»)
- Уверенный, но без «мы лучшие на рынке»
- Можно с лёгкой самоиронией («я давно делаю это, потому что мне самому надоело искать»)
- В сообщениях клиентам — короткими строками, как в чате

### Стиль переносов
Вторая строка — **панчлайн** (короткая, ритмичная, ударно завершает мысль).

Правила:
- Перенос ставить **перед** связкой/новой мыслью: «А пока», «Если», «Когда всё готово», «команда», «Молодожёны заполняют»
- На десктопе пауза часто **после точки** или после **длинного тире** — `<br class="d-only">`
- На мобиле пауза чаще **после тире**, реже после точки — `<br class="m-only">`
- Не отрывать слово от своего предлога: «твоё имя на&nbsp;странице»
- Не оставлять короткое слово сиротой на последней строке

### Сообщения клиентам в Telegram
- Никаких длинных тире (—). Только дефис (-) или двоеточие
- Каждая мысль — отдельная короткая строка
- Цифры писать как есть: 15 000 ₽, не «пятнадцать тысяч»
- Список — нумерованный или через дефис, без украшений
- Максимум один эмодзи в конце, если уместно
- Объяснять ограничения по-человечески: «попробую», «сейчас разберёмся», «жду ответа»

### Запрещено
- «Здравствуйте, уважаемый клиент»
- Восклицательные знаки в каждой фразе («Привет! Это TWOKAIF!»)
- Лозунги-слоганы вроде «Качество, проверенное временем»
- Плотные цепочки эмодзи
- Капс целыми фразами

## 9. Юридика и приватность

### Каждый сайт TWOKAIF
- `<meta name="robots" content="index,follow">` — главные сайты
- `<meta name="robots" content="noindex,nofollow">` — клиентские анкеты, PNG, тестовые домены
- Кук-баннер с тремя кнопками: «Принять все» / «Только необходимые» / «Настройки» (с раскрывашкой по категориям)
- Метрика грузится **только** после явного согласия
- `/privacy` страница со всеми блоками 152-ФЗ
- Реквизиты ИП Тукаева Р. (ИНН 343703190770, ОГРНИП 325344300119216, email rus925@yandex.ru)

### Каждый клиентский домен (`*.twokaif.ru`)
- `noindex, nofollow`
- `robots.txt` с `Disallow: /`
- GitHub-репо приватный
- `.env` с токенами никогда не в git
- В nginx — `X-Robots-Tag: noindex`

### Security headers (везде)
```
strict-transport-security: max-age=63072000; includeSubDomains
x-content-type-options: nosniff
x-frame-options: SAMEORIGIN
referrer-policy: strict-origin-when-cross-origin
permissions-policy: camera=(), microphone=(), geolocation=()
```
